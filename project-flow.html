Smart Bin Project Flow

Sơ đồ chi tiết cho dự án thùng rác thông minh ESP32 + CoAP + Node.js + WebSocket + Firebase. File này mô tả cách dữ liệu cảm biến, lệnh điều khiển web, phân loại rác và thống kê đi qua toàn hệ thống.

1. Tổng Quan Kiến Trúc

HTTP 8080 · WebSocket 8765 · CoAP UDP 5683
Giao diện web Node.js server ESP32 Firebase Phần cứng

2. Sơ Đồ Khối Hoạt Động Từ Đầu Đến Cuối

Power on → kết nối → chọn mode → xử lý rác/lệnh → cập nhật web
Cách đọc sơ đồ: nhánh trái là chế độ AUTO, nhánh phải là MANUAL. Mọi dữ liệu trạng thái cuối cùng đều quay về Node.js server, sau đó server lưu Firebase và đẩy realtime về dashboard bằng WebSocket.

3. Các Thành Phần Chính

Vai trò từng file và từng module

index.html

Dashboard realtime
Hiển thị thống kê rác khô, ướt, kim loại, trạng thái ESP32, cảm biến và lịch sử.
Điều khiển từ web
Gửi lệnh qua WebSocket: mode_auto, mode_manual, open_lid, close_lid, rotate_wet, rotate_dry, rotate_metal.
Nhận dữ liệu
Lắng nghe các event: init, state, telemetry, classify, command_ack, alert.

server.js

HTTP server
Phục vụ dashboard tại http://<server-ip>:8080.
WebSocket server
Nhận lệnh từ web tại cổng 8765, đưa vào hàng đợi pendingCommands.
CoAP server
Nhận/gửi dữ liệu với ESP32 qua UDP cổng 5683.
Firebase
Lưu telemetry, state, events, command history, command ack, alerts và daily stats.

smartbin_esp32_coap_node.ino

AUTO mode
Đọc cảm biến, phát hiện rác, phân loại, xoay stepper tới thùng đúng loại, mở nắp, đóng nắp, quay về home.
MANUAL mode
Nhận lệnh web để mở/đóng nắp và xoay stepper đến thùng khô, ướt hoặc kim loại.
Telemetry
Gửi dữ liệu định kỳ: state, distance, rain, metal, trash_detected và auto_mode.

4. Luồng AUTO Mode

ESP32 tự phát hiện và phân loại rác

→

→
IDLE
1

IDLE

ESP32 ở trạng thái chờ. Nếu đang AUTO, gọi hasTrashMedian() để đo khoảng cách HC-SR04 nhiều mẫu.

2

DETECTED

Khi khoảng cách nhỏ hơn ngưỡng, hệ thống chờ DETECT_WAIT_TIME để rác ổn định.

3

CLASSIFY

Ưu tiên kim loại trước. Nếu không phải kim loại thì kiểm tra cảm biến mưa để xác định rác ướt. Còn lại là rác khô.

4

Gửi kết quả

ESP32 gửi PUT /bin/classify. Server tăng thống kê và broadcast WebSocket cho dashboard.

5

Xoay stepper

Rác khô về home 0°, rác ướt 120°, kim loại 240°.

6

Mở nắp

Servo mở để rác rơi xuống đúng ngăn. Sau DROP_WAIT_TIME, servo đóng lại.

7

Return home

Stepper quay ngược về vị trí gốc để chuẩn bị lần phân loại tiếp theo.

8

WAIT_CLEAR

Chờ cảm biến khoảng cách không còn thấy rác, sau đó quay lại IDLE.

5. Luồng MANUAL Mode

Người dùng điều khiển trên dashboard
1. Người dùng bấm nút
AUTO, MANUAL, Mở nắp, Đóng nắp, Xoay thùng.
→
2. WebSocket
Dashboard gửi JSON: { event: "command", cmd: "rotate_wet" }.
→
3. Server queue
server.js đưa lệnh vào pendingCommands.
4. ESP32 polling
Mỗi COMMAND_CHECK_INTERVAL, ESP32 gọi GET /bin/command.
→
5. Server trả lệnh
CoAP response: { has_command: true, id, cmd }.
→
6. ESP32 thực thi
handleRemoteCommand() gọi servo hoặc handleManualRotate().
7. Gửi ACK
ESP32 gửi PUT /bin/command_ack.
→
8. Server broadcast
Server gửi event command_ack về dashboard.
→
9. UI cập nhật
Dashboard cập nhật log, mode hiện tại và trạng thái cảm biến.

6. Bảng Endpoint Và Event

Các điểm giao tiếp chính trong hệ thống
Đường giao tiếp Nguồn → Đích Payload chính Tác dụng
GET / HTTP Browser → Node.js Không có Tải dashboard index.html.
WS event: command Dashboard → Node.js cmd, device_id, payload Đưa lệnh vào hàng đợi chờ ESP32 lấy.
GET /bin/command ESP32 → Node.js Không có ESP32 hỏi server có lệnh mới không.
PUT /bin/state ESP32 → Node.js s, tt, d, r, m, t, a, msg Cập nhật state machine và mode lên web/Firebase.
PUT /bin/telemetry ESP32 → Node.js s, d, r, m, t, a Gửi dữ liệu cảm biến định kỳ để web biết thiết bị còn online.
PUT /bin/classify ESP32 → Node.js ty, ang, sv, sensor values Cộng thống kê phân loại, lưu event, cập nhật biểu đồ.
PUT /bin/command_ack ESP32 → Node.js cid, cmd, st, msg Xác nhận ESP32 đã nhận/thực hiện lệnh.

7. Lệnh Điều Khiển Và Tác Động

Command từ dashboard đến ESP32
Lệnh Mode yêu cầu ESP32 xử lý Kết quả mong đợi
mode_auto Bất kỳ setMode(MODE_AUTO), đóng nắp, stepper home. ESP32 tự động đọc cảm biến và phân loại rác.
mode_manual Bất kỳ setMode(MODE_MANUAL), đóng nắp, stepper home. ESP32 dừng auto classify, chờ lệnh thủ công.
open_lid Bất kỳ openLid() Servo mở nắp.
close_lid Bất kỳ closeLid() Servo đóng nắp.
rotate_dry Manual, hoặc tự chuyển sang Manual moveStepperTo(0) Stepper về thùng rác khô/home.
rotate_wet Manual, hoặc tự chuyển sang Manual moveStepperTo(WET_STEPS) Stepper xoay đến vị trí rác ướt khoảng 120°.
rotate_metal Manual, hoặc tự chuyển sang Manual moveStepperTo(METAL_STEPS) Stepper xoay đến vị trí kim loại khoảng 240°.

8. Mapping Phần Cứng ESP32

Các chân đang dùng trong sketch
Thiết bị Chân ESP32 Ghi chú
HC-SR04 TRIG 18 Xuất xung trigger.
HC-SR04 ECHO 19 Đọc xung echo. Nên hạ áp ECHO nếu module xuất 5V.
Cảm biến kim loại 2 Dùng INPUT_PULLUP, cấu hình active low.
Cảm biến mưa analog 0 Đọc analog để phân biệt rác ướt.
Thiết bị Chân ESP32 Ghi chú
Servo nắp 13 Đóng/mở nắp. Nên cấp nguồn riêng đủ dòng.
ULN2003 IN1 25 Stepper 28BYJ-48.
ULN2003 IN2 26 Stepper 28BYJ-48.
ULN2003 IN3 27 Stepper 28BYJ-48.
ULN2003 IN4 14 Stepper 28BYJ-48.
Quan trọng: không dùng GPIO 6-11 trên nhiều board ESP32 vì các chân này thường nối với SPI flash. Nguồn 5V cho servo/stepper nên tách riêng với ESP32, nhưng phải nối chung GND.

9. Checklist Debug Theo Luồng

Dùng khi web không điều khiển được ESP32 hoặc stepper không xoay
A

WebSocket

  • Dashboard phải hiện WebSocket đã kết nối.
  • Khi bấm nút, log phải có Server đã nhận lệnh.
  • Server console phải in [WS MESSAGE].
B

Server queue

  • Lệnh phải vào pendingCommands.
  • Khi ESP32 hỏi, console phải in [CoAP COMMAND] ESP32 get command.
  • Kiểm tra ESP32 có đúng IP server trong sketch.
C

CoAP

  • ESP32 phải gửi telemetry định kỳ.
  • Server phải nhận PUT /bin/telemetry.
  • ESP32 phải gọi GET /bin/command.
D

ESP32 command

  • Serial Monitor phải có [CMD] rotate_wet.
  • Khi xoay phải có Stepper current=... desired=... delta=....
  • Nếu có log nhưng motor không quay, kiểm tra nguồn và dây ULN2003.
